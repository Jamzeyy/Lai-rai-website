Executive Summary

The Lai Rai Vietnamese Tapas website has been designed and developed to conform to the Web Content Accessibility Guidelines (WCAG) 2.2 at the AA conformance level. WCAG 2.2 was published as a W3C Recommendation on October 5, 2023, and is the current international standard for web accessibility.

This report documents how each applicable success criterion has been addressed, ensuring the website is usable by everyone, including people who are blind, have low vision, are deaf or hard of hearing, have mobility impairments, have cognitive or learning disabilities, or rely on assistive technologies such as screen readers, keyboard-only navigation, or voice control.

Implementation Details — WCAG 2.2 New Criteria

2.4.11 Focus Not Obscured (Minimum) Pass

When any UI component receives keyboard focus, it is not entirely hidden by author-created content such as the fixed navigation bar.

How We Comply

  • scroll-padding-top: 80px on the <html> element ensures anchored scroll destinations clear the fixed navigation
  • scroll-margin-top: 80px applied to section targets and focused elements
  • A JavaScript focusin event listener detects if a focused element is behind the fixed navigation and automatically scrolls it into full view
  • All focused elements receive z-index: 1; position: relative to prevent overlapping by other content

2.4.12 Focus Not Obscured (Enhanced) Pass

No part of the focus indicator is hidden by author-created content.

How We Comply

  • Focus indicators use outline-offset: 4px, placing the indicator outside the element boundary so no surrounding content can overlap it
  • The automatic scroll correction JavaScript ensures the entirety of the focused element (plus its focus ring) is visible above the fixed navigation

2.4.13 Focus Appearance Pass

Focus indicators have sufficient size and contrast to be clearly visible.

How We Comply

  • Focus outline: 3px solid #9E5A3C (terracotta) with 4px offset
  • The 3px outline on all sides exceeds the minimum 2px perimeter thickness required
  • Contrast ratio of #9E5A3C against #F7F3EB (cream background) = 3.8:1, exceeding the 3:1 minimum for focus indicators
  • On dark backgrounds (location section), buttons receive custom focus outlines with var(--gold) to maintain contrast

2.5.7 Dragging Movements N/A

The website contains no functionality that uses dragging movements. All interactions are achieved through simple clicks, taps, or keyboard activation.

2.5.8 Target Size (Minimum) Pass

All interactive targets meet or exceed the 24×24 CSS pixel minimum size requirement.

How We Comply

  • Navigation links: min-height: 24px with padding: 0.5rem, effective size exceeds 44×24px
  • Hamburger menu button: min-width: 44px; min-height: 44px
  • CTA buttons: min-height: 44px with generous horizontal padding
  • Menu category filter buttons: min-height: 44px; min-width: 44px
  • Social media links: min-width: 44px; min-height: 44px
  • Scroll indicator: min-width: 44px; min-height: 44px
  • Footer links: min-height: 44px with padding
  • All targets maintain minimum sizing at mobile breakpoints (480px and below)

3.2.6 Consistent Help N/A

This is a single-page website. The criterion requires consistency across multiple pages, which does not apply. The ordering link and contact information appear consistently in the navigation, main content, and footer.

3.3.7 Redundant Entry N/A

Additional Accessibility Features

Beyond WCAG 2.2 requirements, the website includes the following enhancements:

Reduced Motion Support

The @media (prefers-reduced-motion: reduce) query disables all animations and transitions for users who experience motion sensitivity or vestibular disorders. Scroll-reveal elements appear immediately without animation.

High Contrast Mode

The @media (prefers-contrast: high) query activates an enhanced color scheme with pure white backgrounds, darker text (#000000), bolder borders (2px), and wider focus indicators (4px) for users with low vision.

Security for External Links

All external links (target="_blank") include rel="noopener noreferrer" to prevent tab-napping vulnerabilities, plus screen-reader text indicating a new tab will open.

Semantic HTML5

The site uses proper semantic elements: <nav> (with distinct aria-labels for main and footer navigation), <main>, <section>, <article>, <address>, <blockquote>, <table>, and <footer>. This ensures assistive technologies can accurately convey page structure.

Tap-to-Call Phone Number

The phone number uses a tel: link, allowing mobile users and assistive technologies to initiate a phone call directly.

Testing Methodology

Accessibility was evaluated using the following approach:

  • Manual keyboard testing — Tab navigation through all interactive elements, verifying focus visibility and logical order
  • Screen reader compatibility — Structural landmarks, heading hierarchy, ARIA live regions, and Vietnamese language tags verified
  • Color contrast analysis — All text/background combinations tested against WCAG AA (4.5:1) and non-text contrast (3:1) requirements
  • Target size measurement — All interactive elements verified to meet 24×24px minimum (most exceed at 44×44px)
  • Responsive testing — Accessibility features validated across viewport widths from 320px to 1920px
  • Reduced motion & high contrast — Tested with OS-level preferences enabled
